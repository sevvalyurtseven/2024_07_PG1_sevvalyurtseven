Chapter 3 Summary

Control Statements and Operators

  • A boolean type variable (type variable) can store a true or false value.
  • The relational operators (relational operators) (<, <=, ==, !=, >, >=) yield a Boolean value.
  • Selection statements (selection statements) are used for programming with alternative courses of action. There are several types of selection statements:
    One-way if statements (One-way if statements),
    Two-way if-else statements (Two-way if-else statements),
    Nested if statements (Nested if statements),
    Multi-way if-else statements (Multi-way if-else statements),
    Switch statements (Switch statements), and
    Conditional expressions (Conditional expressions).
  • The various if statements (if statements) all make control decisions based on a Boolean expression (Boolean expression). Based on the true (true) or false (false) evaluation of the expression, these statements take one of two possible courses.
  • The Boolean operators (Boolean operators) &&, ||, !, and ^ operate with Boolean values (Boolean values) and variables (variables).
  • When evaluating p1 && p2 (evaluating p1 && p2), Java first evaluates p1 and then evaluates p2 if p1 is true; if p1 is false, it does not evaluate p2.

    When evaluating p1 || p2 (evaluating p1 || p2), Java first evaluates p1 and then evaluates p2 if p1 is false; if p1 is true, it does not evaluate p2.

    Therefore, && is referred to as the conditional or short-circuit AND operator (AND operator), and || is referred to as the conditional or short-circuit OR operator (OR operator).
  • The switch statement (switch statement) makes control decisions based on a switch expression (expression) of type char, byte, short, int, or String.
  • The keyword break (keyword) is optional in a switch statement (switch statement), but it is normally used at the end of each case in order to skip the remainder (remainder) of the switch statement. If the break statement is not present, the next case statement will be executed (executed).
  • The operators (operators) in expressions (expressions) are evaluated in the order determined by the rules of parentheses (parentheses), operator precedence (operator precedence), and operator associativity (operator associativity).
  • Parentheses (Parentheses) can be used to force the order of evaluation (evaluation) to occur in any sequence.
  • Operators with higher precedence (precedence) are evaluated earlier. For operators of the same precedence, their associativity (associativity) determines the order of evaluation.
  • All binary operators (binary operators) except assignment operators (assignment operators) are left-associative (left-associative); assignment operators (assignment operators) are right-associative (right-associative).
