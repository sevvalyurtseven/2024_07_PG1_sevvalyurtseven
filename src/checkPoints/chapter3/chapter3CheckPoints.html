Question:

  • boolean b = true;
  • int i = (int) b;
  • int i = 1;
  • boolean b = (boolean) i;

Answer:

Explanation: In Java, direct casting between boolean and int is not allowed because they represent fundamentally different data types. Here's why:

1. boolean b = true;
This is a valid statement. A boolean variable can be assigned the value true or false.

2. int i = (int) b;
This line will cause a compilation error. In Java, boolean and int are not directly compatible types. boolean is typically used for logical values (true or false), while int is used for numerical values. There is no direct conversion between these types.

3. int i = 1;
This is a valid statement. An int variable can be assigned any integer value.

4. boolean b = (boolean) i;
This line will also cause a compilation error. Similar to the second statement, boolean and int are not directly compatible. You cannot cast an integer directly to a boolean.

Answer:

1. Data Types for a Switch Variable:
- In Java, the switch variable can be of the following types: int, char, byte, short, enum, String, and Integer wrapper types.

2. What happens if break is not used?
- If the keyword break is not used after a case is processed, the execution will "fall through" to the next case, meaning the subsequent case(s) will be executed regardless of their conditions.

3. Converting Between Switch and If-Else:
- Yes, you can convert a switch statement to an equivalent if-else statement and vice versa. However, a switch statement is generally more concise and easier to read when there are multiple conditions based on a single variable.

4. Advantages of Using a Switch Statement:
- Switch statements can be more efficient than if-else chains because they are often implemented using a jump table in the background. - They improve code readability when checking a single variable against multiple values.

Precedence Order of Boolean Operators:

  • 1. `!` (Logical NOT)
  • 2. `&&` (Logical AND)
  • 3. `||` (Logical OR)

Expression Evaluations:


true || true && false

// Evaluate `true && false` first (AND has higher precedence):
// true && false => false
// Now evaluate `true || false`:
// true || false => true
// Final Result: true

true && true || false

// Evaluate `true && true` first (AND has higher precedence):
// true && true => true
// Now evaluate `true || false`:
// true || false => true
// Final Result: true

Answer: True. All binary operators except the assignment operator (`=`) are left associative. This means that in expressions with multiple operators of the same precedence, evaluation occurs from left to right.


2 * 2 - 3 > 2 && 4 - 2 > 5

// Evaluate `2 * 2`:
// 2 * 2 = 4
// Now evaluate `4 - 3`:
// 4 - 3 = 1
// Check `1 > 2`:
// 1 > 2 => false
// Next, evaluate `4 - 2`:
// 4 - 2 = 2
// Check `2 > 5`:
// 2 > 5 => false
// Final result is `false && false` => false

2 * 2 - 3 > 2 || 4 - 2 > 5

// Evaluate `2 * 2`:
// 2 * 2 = 4
// Now evaluate `4 - 3`:
// 4 - 3 = 1
// Check `1 > 2`:
// 1 > 2 => false
// Next, evaluate `4 - 2`:
// 4 - 2 = 2
// Check `2 > 5`:
// 2 > 5 => false
// Final result is `false || false` => false

Answer:

  • 1. `(x > 0 && x < 10)` is the same as `((x > 0) && (x < 10))`
    - Yes, they are equivalent because the order of evaluation is the same whether the parentheses are present or not.
  • 2. `(x > 0 || x < 10)` is the same as `((x > 0) || (x < 10))`
    - Yes, they are equivalent because the order of evaluation is the same whether the parentheses are present or not.
  • 3. `(x > 0 || x < 10 && y < 0)` is the same as `(x > 0 || (x < 10 && y < 0))`
    - No, they are not the same. Due to the precedence of operators, `&&` is evaluated before `||`. Therefore, `x > 0 || x < 10 && y < 0` evaluates to `x > 0 || (x < 10 && y < 0)` by default. Explicit parentheses are required to change the grouping.
