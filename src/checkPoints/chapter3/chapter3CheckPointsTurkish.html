Soru:

Cevap:

Açıklama: Java'da, boolean ve int arasında doğrudan dönüştürme yapılmasına izin verilmez çünkü bu veri tipleri temelde farklıdır. İşte nedenleri:

1. boolean b = true;
Bu geçerli bir ifadedir. Bir boolean değişkenine true veya false değeri atanabilir.

2. int i = (int) b;
Bu satır derleme hatasına neden olur. Java'da, boolean ve int doğrudan uyumlu veri türleri değildir. boolean genellikle mantıksal değerler (true veya false) için kullanılırken, int sayısal değerler için kullanılır. Bu türler arasında doğrudan bir dönüşüm yoktur.

3. int i = 1;
Bu geçerli bir ifadedir. Bir int değişkenine herhangi bir tamsayı değeri atanabilir.

4. boolean b = (boolean) i;
Bu satır da derleme hatasına neden olur. İkinci ifadeye benzer şekilde, boolean ve int doğrudan uyumlu değildir. Bir tam sayıyı doğrudan bir boolean'a dönüştüremezsiniz.

Cevap:

1. switch Değişkeni İçin Veri Tipleri:
- Java'da switch değişkeni aşağıdaki tiplerde olabilir: int, char, byte, short, enum, String ve Integer sarmalayıcı tipler.

2. break Kullanılmazsa Ne Olur?
- Eğer break anahtarı bir case işleminden sonra kullanılmazsa, yürütme bir sonraki case'e "düşer", yani koşullara bakılmaksızın sonraki case(ler) çalıştırılır.

3. switch ile if-else Arasında Dönüştürme:
- Evet, bir switch ifadesini eşdeğer bir if-else ifadesine ve tam tersine dönüştürebilirsiniz. Ancak, bir switch ifadesi genellikle tek bir değişkene dayalı birden fazla koşul olduğunda daha özlü ve okunması daha kolaydır.

4. switch İfadesi Kullanmanın Avantajları:
- Switch ifadeleri, if-else zincirlerine göre daha verimli olabilir çünkü arka planda genellikle bir atlama tablosu kullanılarak uygulanır. - Tek bir değişkeni birden fazla değerle karşılaştırırken kodun okunabilirliğini artırır.

Boolean Operatörlerinin Öncelik Sırası:

  • 1. `!` (Mantıksal DEĞİL)
  • 2. `&&` (Mantıksal VE)
  • 3. `||` (Mantıksal VEYA)

İfade Değerlendirmeleri:


true || true && false

// Önce `true && false` değerlendirilir (VE daha yüksek önceliğe sahiptir):
// true && false => false
// Şimdi `true || false` değerlendirilir:
// true || false => true
// Nihai Sonuç: true

true && true || false

// Önce `true && true` değerlendirilir (VE daha yüksek önceliğe sahiptir):
// true && true => true
// Şimdi `true || false` değerlendirilir:
// true || false => true
// Nihai Sonuç: true

Cevap: Doğru. Atama operatörü (`=`) dışındaki tüm ikili operatörler soldan birleşimlidir. Bu, aynı önceliğe sahip birden fazla operatörle yapılan ifadelerde değerlendirmenin soldan sağa doğru yapıldığı anlamına gelir.


2 * 2 - 3 > 2 && 4 - 2 > 5

// `2 * 2` değerlendirilir:
// 2 * 2 = 4
// Şimdi `4 - 3` değerlendirilir:
// 4 - 3 = 1
// `1 > 2` kontrol edilir:
// 1 > 2 => false
// Sonra `4 - 2` değerlendirilir:
// 4 - 2 = 2
// `2 > 5` kontrol edilir:
// 2 > 5 => false
// Nihai sonuç `false && false` => false

2 * 2 - 3 > 2 || 4 - 2 > 5

// `2 * 2` değerlendirilir:
// 2 * 2 = 4
// Şimdi `4 - 3` değerlendirilir:
// 4 - 3 = 1
// `1 > 2` kontrol edilir:
// 1 > 2 => false
// Sonra `4 - 2` değerlendirilir:
// 4 - 2 = 2
// `2 > 5` kontrol edilir:
// 2 > 5 => false
// Nihai sonuç `false || false` => false

Cevap:

  • 1. `(x > 0 && x < 10)` ifadesi `((x > 0) && (x < 10))` ile aynı mı?
    - Evet, aynı çünkü parantezler olsun olmasın değerlendirme sırası aynıdır.
  • 2. `(x > 0 || x < 10)` ifadesi `((x > 0) || (x < 10))` ile aynı mı?
    - Evet, aynı çünkü parantezler olsun olmasın değerlendirme sırası aynıdır.
  • 3. `(x > 0 || x < 10 && y < 0)` ifadesi `(x > 0 || (x < 10 && y < 0))` ile aynı mı?
    - Hayır, aynı değiller. Operatörlerin önceliği nedeniyle `&&` `||`'dan önce değerlendirilir. Bu yüzden varsayılan olarak `(x > 0 || x < 10 && y < 0)` ifadesi `(x > 0 || (x < 10 && y < 0))` olarak değerlendirilir. Gruplamayı değiştirmek için açık parantezler gereklidir.
