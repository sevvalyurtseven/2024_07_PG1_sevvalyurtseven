Cevap:

25 / 4 işleminin sonucu 6'dır (her iki operand tamsayı olduğundan tam sayı bölmesi yapılır).

Kayan nokta sonucu elde etmek için ifadeyi 25.0 / 4 veya 25 / 4.0 olarak yazabilirsiniz; bu, 6.25 sonucunu verir. Bunun nedeni, operandlardan en az birinin kayan nokta sayısı olmasıdır, bu nedenle bölme kayan nokta bölmesi olarak yapılır. 


System.out.println(2 * (5 / 2 + 5 / 2));  // Çıktı: 8
// (5 / 2) 2'dir (tamsayı bölmesi), bu nedenle (2 + 2) 4'tür ve 2 * 4 = 8.

System.out.println(2 * 5 / 2 + 2 * 5 / 2); // Çıktı: 10
// 2 * 5 10'dur, 10 / 2 5'tir. Diğer 2 * 5 / 2, başka bir 5 verir. 5 + 5 = 10.

System.out.println(2 * (5 / 2));          // Çıktı: 4
// (5 / 2) 2'dir (tamsayı bölmesi), bu nedenle 2 * 2 = 4.

System.out.println(2 * 5 / 2);            // Çıktı: 5
// 2 * 5 10'dur, 10 / 2 5'tir.
        

System.out.println("25 / 4 is " + 25 / 4);         // Çıktı: 25 / 4 is 6
// Tamsayı bölmesi, sonuç 6.

System.out.println("25 / 4.0 is " + 25 / 4.0);     // Çıktı: 25 / 4.0 is 6.25
// Kayan nokta bölmesi, sonuç 6.25.

System.out.println("3 * 2 / 4 is " + 3 * 2 / 4);   // Çıktı: 3 * 2 / 4 is 1
// 3 * 2 6'dır, 6 / 4 1'dir (tamsayı bölmesi).

System.out.println("3.0 * 2 / 4 is " + 3.0 * 2 / 4); // Çıktı: 3.0 * 2 / 4 is 1.5
// 3.0 * 2 6.0'dır, 6.0 / 4 1.5'tir (kayan nokta bölmesi).

Şu anki saniye, dakika ve saati Java'da elde etmek için, Java 8 Tarih ve Saat API'sinin bir parçası olan java.time.LocalTime sınıfını kullanabilirsiniz. İşte bunu nasıl yapabileceğiniz:


import java.time.LocalTime;

public class CurrentTimeExample {
    public static void main(String[] args) {
        // Mevcut zamanı al
        LocalTime currentTime = LocalTime.now();

        // Mevcut saniye, dakika ve saati çıkar

        System.out.println("Current second: " + currentSecond);
    }
}

Cevap:

Tamsayı Taşması (Integer Overflow): Tamsayı taşması, bir aritmetik işlemin, tamsayı türü tarafından temsil edilebilecek aralığı aşması durumunda meydana gelir. Örneğin, bir int türünün tutabileceği maksimum değere 1 eklerseniz, bu minimum değere döner.

Kayan Nokta Taşması (Floating-Point Overflow): Evet, kayan nokta işlemleri taşmaya neden olabilir. Bu, bir hesaplamanın, bir float veya double tarafından temsil edilebilecek en büyük değeri aşması durumunda olur. Sonuç genellikle pozitif veya negatif sonsuzluk olarak temsil edilir.

Cevap:

Hayır, taşma Java'da çalışma zamanı hatasına neden olmaz. Bunun yerine, değer sarılır (wraps around) (tamsayı türleri için) veya sonsuzluğa dönüşür (kayan nokta türleri için). Ancak, bu programda beklenmeyen davranışlara ve mantık hatalarına yol açabilir.

Cevap:

Yuvarlama Hatası (Round-Off Error): Bir sayının, veri türünün doğruluk sınırlamaları nedeniyle daha kısa veya daha az kesin bir değere yuvarlanması durumunda oluşur.

Tamsayı İşlemleri: Tamsayı işlemleri yuvarlama hatalarına neden olmaz çünkü bunlar veri türünün aralığı içinde tamdır.

Kayan Nokta İşlemleri: Evet, kayan nokta işlemleri yuvarlama hatalarına neden olabilir çünkü bilgisayarlarda kayan nokta sayıların temsil edildiği sınırlı doğruluk nedeniyle hesaplamalar sırasında küçük farklar yuvarlanabilir ve bu da doğruluğun azalmasına yol açabilir.
