ASCII Kodu: ASCII (American Standard Code for Information Interchange), her karaktere bir sayısal değer atayan bir karakter kodlama standardıdır. Örneğin, 'A' karakteri ASCII kodu olarak 65'e sahiptir.

ASCII Kodundan Karakter: Java'da, bir tamsayıyı (ASCII kodu) tür dönüştürme kullanarak karşılık gelen karaktere dönüştürebilirsiniz. Örneğin, (char)65 size 'A' karakterini verir.

Aşağıdaki Java print komutları, belirli karakterler için ASCII kodlarını bulmayı ve ilgili ASCII kodlarından karakterleri geri almayı nasıl yapacağınızı gösterir.

Cevap:

System.out.println((int)'A'); // Çıktı: 65
System.out.println((int)'B'); // Çıktı: 66
System.out.println((int)'a'); // Çıktı: 97
System.out.println((int)'b'); // Çıktı: 98

System.out.println((char)40); // Çıktı: (
System.out.println((char)59); // Çıktı: ;
System.out.println((char)79); // Çıktı: O
System.out.println((char)85); // Çıktı: U
System.out.println((char)90); // Çıktı: Z

System.out.println((char)0x40); // Çıktı: @
System.out.println((char)0x5A); // Çıktı: Z
System.out.println((char)0x71); // Çıktı: q
System.out.println((char)0x72); // Çıktı: r
System.out.println((char)0x7A); // Çıktı: z

Java'da, `printf` methodunda farklı veri türlerini formatlamak için format specifiers kullanılır. Farklı veri türleri için format specifiers şunlardır:

- Boolean değeri için `%b`
- Karakter için `%c`
- Ondalık tamsayı için `%d`
- Floating-point sayı için `%f`
- String için `%s`

Cevap:

- Boolean: `%b`
- Karakter: `%c`
- Ondalık Tamsayı: `%d`
- Floating-point Sayı: `%f`
- String: `%s`

Açıklama:

Bu ifadeler, Java'da `printf` methodunu kullanarak formatlanmış çıktı üretir. Bu ifadelerdeki hatalar, format specifiers ile geçirilen argümanların sayısı ve türleriyle ilgilidir. Format stringinde her format specifier, doğru türde karşılık gelen bir argüman bekler.

Her ifadeyi değerlendirelim:

Cevap:

(a) System.out.printf("%5d %d", 1, 2, 3); - Yanlış (Çok fazla argüman var; iki format specifier var ama üç argüman var)
(b) System.out.printf("%5d %f", 1); - Yanlış (Yeterli argüman yok; iki format specifier var ama bir argüman var)
(c) System.out.printf("%5d %f", 1, 2); - Yanlış (Tür uyumsuzluğu; ikinci argüman bir tamsayı ama `%f` floating-point sayı bekliyor)

Açıklama:

Bu soru, Java'da `printf` ifadelerinin çıktısını değerlendirmemizi ister. Format specifiers ve sağlanan argümanlar, nihai çıktıyı belirler.

Her ifadenin çıktısını değerlendirelim:

Cevap:

(a) System.out.printf("amount is %f %e\n", 32.32, 32.32);
Çıktı: `amount is 32.320000 3.232000e+01` (Floating-point sayı hem standart hem de bilimsel notasyonda gösterilir)

(b) System.out.printf("amount is %5.2%% %5.4e\n", 32.327, 32.32);
Çıktı: `amount is 32.33% 3.2320e+01` (Yüzde işareti `%%` ile kaçırılır ve floating-point sayı 2 ondalık basamakla yuvarlanır ve gösterilir)

(c) System.out.printf("%6b\n", (1 > 2));
Çıktı: ` false` (Boolean değer, 6 karakter genişliğinde sağa yaslanır)

(d) System.out.printf("%6s\n", "Java");
Çıktı: ` Java` (String "Java" 6 karakter genişliğinde sağa yaslanır)

(e) System.out.printf("%-6b%s\n", (1 > 2), "Java");
Çıktı: `false Java` (Boolean değer, 6 karakter genişliğinde sola yaslanır, ardından "Java" stringi gelir)

(f) System.out.printf("%6b%-8s\n", (1 > 2), "Java");
Çıktı: ` falseJava ` (Boolean değer, 6 karakter genişliğinde sağa yaslanır, ardından "Java" stringi 8 karakter genişliğinde sola yaslanır)
